# UX_PRINCIPLES.md

## Design direction

The app should feel like a quiet utility, not a workspace. It exists on the side of the user's workflow and appears only when needed.

## Principles

* Minimal and modern
* Fast to understand
* Low visual noise
* Immediate interaction
* Short paths to action
* Clear state around expiry

## Interface guidance

* Compact floating panel
* Soft, clean visual style
* Strong readability
* Clear drag-and-drop affordances
* Preview where useful, not everywhere
* Avoid deep navigation or multi-page complexity

## Primary UI sections

* Header with app name or minimal identity
* Add/import actions
* Search field
* Active shelf items list
* Optional settings access

## Item card guidance

Each item should display:

* Name
* Type or extension
* Size
* Added time or expires in
* Preview thumbnail when relevant
* Quick actions
